컴파일

Sass 컴파일은 를 사용하는 방법과 2. node + gulp 를 사용하는 방법 두 가지 중 편리한 것을 사용한다.

① VSCODE > 마켓플레이스 : "Live Sass Compiler" 설치

② 확장설정 (settings.json 편집) - 경로, 컴파일 형태 수정

	"liveSassCompile.settings.formats": [
	// This is Default.
	{
		"format": "compressed",
		"extensionName": ".css",
		"savePath": "~/../css/"
	},
],
"liveSassCompile.settings.includeItems": [
	"./data/skin/assets/*/scss/*.scss",
],
"liveSassCompile.settings.autoprefix": ["> 1%", "last 2 versions"],
"liveSassCompile.settings.generateMap": false,

③ VSCODE "watch sass" 클릭 시 자동 컴파일 시작

2. node + gulp

① Node.js 설치

Node.js 설치 (안정적, 신뢰도 높음 설치)

② Gulp.js 설치

Gulp.js 설치

  1. node, npm, npx 설치 확인 
    node --version
npm --version
npx --version
  2. gulp-cli 설치 : Gulp를 콘솔에서 편리하게 실행할 수 있게 해주는 유틸리티 도구 
    npm install gulp-cli -g
  3. 경로 이동 
    cd C:\프로젝트명
  4. 프로젝트 폴더에 package.json 생성 
    npm init
  5. gulp package 설치 (node_modules 폴더 생성됨) 
    npm install gulp --save-dev
  6. gulp 버전 확인 
    gulp --version
  7. gulpfile.js 생성 
    function defaultTask(cb) {
// place code for your default task here
cb();
}

exports.default = defaultTask
  8. gulp 실행 
    gulp

  • gulp-sourcemaps : css map 생성해줌

    source map : 배포용으로 빌드한 파일(ex. css)과 원본파일(ex. scss)을 연결해줌.

  • gulp-sass : sass 컴파일
  • browser-sync : 파일 변경 시 자동 refresh, 여러 브라우저를 띄어놓고, 한 브라우저에서 클릭 동작 시 동일한 처리
  • gulp-autoprefixer : 최신 CSS를 IE나 구형 브라우저가 이해할 수 있게 prefix 생성해줌.

    ↓ 예시

    -webkit-box-pack: center;
-ms-flex-pack: center;
justify-content: center;
-webkit-box-align: center;
-ms-flex-align: center;
align-items: center;
-webkit-box-orient: vertical;
-webkit-box-direction: normal;
-ms-flex-direction: column;
  • gulp-headerfooter : html include
  • gulp-remove-empty-lines : 빈줄 삭제
  • gulp-inject : 경로에 따른 include 추가
  • gulp-ejs : ejs → html 파일 변환
  • gulp-rename : 병합, 압축하는 파일을 이름을 변경하여 출력 (압축 x파일과 압축 ㅇ파일 두개 출력 → 사용 예시 : min.js - js)
  • gulp-imagemin : 이미지 압축
  • gulp-concat : 파일 이어 붙이기
  • gulp-uglify : 파일 압축
  • gulp-babel : es6 → es5 변환
  • gulp-plumber : 에러로그 출력
